# SIKAPA Production Deployment Guide\n\n## 🎯 Before You Commit\n\n### 1. Environment Configuration\nEnsure all environment variables are set correctly:\n\n```bash\n# Root .env file\nNODE_ENV=production\nPORT=3000\nFIREBASE_API_KEY=xxxxxxxxxxxx\nFIREBASE_PROJECT_ID=sikapa-xxxxx\nFIREBASE_AUTH_DOMAIN=xxxxxxxxxxxx\nFIREBASE_DATABASE_URL=xxxxxxxxxxxx\n```\n\n### 2. Code Review Checklist\n\n```bash\n# Check for console logs\ngrep -r \"console\\.log\" --include=\"*.js\" --include=\"*.html\" backend/ | grep -v \"node_modules\"\n\n# Check for debug code\ngrep -r \"debugger\" --include=\"*.js\" backend/\n\n# Check for TODO comments\ngrep -r \"TODO\\|FIXME\\|HACK\" --include=\"*.js\" backend/ | grep -v \"node_modules\"\n```\n\n---\n\n## 📦 Commit Process\n\n### Step 1: Test Everything Locally\n```bash\n# Start the server\ncd backend\nnode server.js\n\n# In another terminal, test critical endpoints\ncurl http://localhost:3000/api/health\ncurl http://localhost:3000/api/user/stats/test-user-id\n```\n\n### Step 2: Prepare Commit\n```bash\n# Check what changed\ngit status\n\n# Stage files\ngit add .\n\n# Verify changes before commit\ngit diff --staged\n```\n\n### Step 3: Create Meaningful Commit\n```bash\ncommit message format: \"[FEATURE/FIX/REFACTOR] Brief description\n\n# Examples:\ngit commit -m \"[FEATURE] Complete mobile optimization for all pages\n- Added comprehensive media queries for 320px-1920px screens\n- Implemented touch-friendly button sizes (44x44px minimum)\n- Enhanced viewport meta tags for iOS/Android\n- Optimized typography for mobile viewing\n- Fixed scrolling issues on Tasks and Me tabs\"\n\ngit commit -m \"[FIX] Correct balance calculations\n- Fixed totalBalance = account balance + withdrawable rewards\n- Fixed todayEarning to include tier claims\n- Cleared test transaction data\n- Verified all stats endpoints return correct values\"\n```\n\n### Step 4: Push to Repository\n```bash\n# For first-time push\ngit push -u origin main\n\n# Subsequent pushes\ngit push origin main\n```\n\n---\n\n## 🚀 Deployment to Production\n\n### Method 1: Manual Deployment\n\n```bash\n# 1. SSH into production server\nssh user@production-server.com\n\n# 2. Navigate to app directory\ncd /var/www/sikapa\n\n# 3. Pull latest code\ngit pull origin main\n\n# 4. Install/update dependencies\ncd backend\nnpm install --production\n\n# 5. Backup existing files\ncp -r logs logs.backup.$(date +%Y%m%d%H%M%S)\n\n# 6. Start/restart server\npm2 restart sikapa-server\n# or\nnpm start\n\n# 7. Verify server is running\ncurl http://localhost:3000/api/health\n\n# 8. Monitor logs\ntail -f logs/sikapa.log\n```\n\n### Method 2: Using PM2 (Recommended)\n\n```bash\n# 1. Install PM2 globally (if not already installed)\nnpm install -g pm2\n\n# 2. Create PM2 ecosystem file\ncat > ecosystem.config.js << 'EOF'\nmodule.exports = {\n  apps: [{\n    name: 'sikapa-server',\n    script: './server.js',\n    cwd: './backend',\n    env: {\n      NODE_ENV: 'production',\n      PORT: 3000\n    },\n    error_file: './logs/sikapa-error.log',\n    out_file: './logs/sikapa-out.log',\n    log_date_format: 'YYYY-MM-DD HH:mm:ss Z',\n    merge_logs: true,\n    autorestart: true,\n    watch: false,\n    max_memory_restart: '500M',\n    max_restarts: 10\n  }]\n};\nEOF\n\n# 3. Start/restart with PM2\npm2 start ecosystem.config.js --name sikapa-server\npm2 save\npm2 startup\n\n# 4. Monitor\npm2 status\npm2 logs sikapa-server\n```\n\n### Method 3: Using Docker (Advanced)\n\n```bash\n# 1. Create Dockerfile\ncat > Dockerfile << 'EOF'\nFROM node:16-alpine\nWORKDIR /app\nCOPY package*.json ./\nRUN npm install --production\nCOPY . .\nEXPOSE 3000\nCMD [\"npm\", \"start\"]\nEOF\n\n# 2. Build image\ndocker build -t sikapa-server:latest .\n\n# 3. Run container\ndocker run -d \\\n  --name sikapa-server \\\n  -p 3000:3000 \\\n  -e NODE_ENV=production \\\n  -v $(pwd)/logs:/app/logs \\\n  sikapa-server:latest\n\n# 4. Monitor\ndocker logs -f sikapa-server\n```\n\n---\n\n## ✅ Post-Deployment Verification\n\n```bash\n# 1. Check server health\ncurl https://your-domain.com/api/health\n\n# 2. Test API endpoints\ncurl https://your-domain.com/api/user/stats/test-user\n\n# 3. Check for errors\ngrep -i \"error\" logs/sikapa.log | tail -20\n\n# 4. Monitor performance\nwatch -n 5 'ps aux | grep node'\n\n# 5. Check with browser\n# Navigate to https://your-domain.com and test:\n# - Login functionality\n# - Dashboard loading\n# - Tier claims\n# - Balance updates\n# - Navigation between tabs\n```\n\n---\n\n## 🔄 Rollback Procedure\n\nIf something goes wrong:\n\n```bash\n# 1. Stop the server\npm2 stop sikapa-server\n# or\nnpm kill 3000\n\n# 2. Revert to previous version\ngit revert HEAD\n# or\ngit reset --hard previous-commit-hash\n\n# 3. Restore backup\nrm -rf logs\ncp -r logs.backup.TIMESTAMP logs\n\n# 4. Restart server\npm2 start sikapa-server\n# or\nnpm start\n\n# 5. Verify rollback successful\ncurl http://localhost:3000/api/health\n\n# 6. Notify team\necho \"Rollback to previous version completed\"\n```\n\n---\n\n## 📊 Monitoring Dashboard\n\n### Key Metrics to Monitor\n- **Server Response Time**: Should be < 500ms\n- **Error Rate**: Should be < 0.1%\n- **CPU Usage**: Should be < 60%  \n- **Memory Usage**: Should be < 70%\n- **Database Connections**: Should be stable\n\n### Useful Commands\n\n```bash\n# Monitor in real-time\nwatch -n 1 'curl -s http://localhost:3000/api/health | json_pp'\n\n# Check running processes\nps aux | grep node\nps aux | grep npm\n\n# Monitor memory\nfree -h\n\n# Check disk space\ndf -h\n\n# View logs\ntail -f logs/sikapa.log\n\n# Search logs\ngrep \"error\" logs/sikapa.log\n```\n\n---\n\n## 🔐 Security Checks Pre-Launch\n\n```bash\n# 1.
